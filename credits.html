Security considerations

When dealing with user information it is extremely important to consider security problems of the website. If the website would allow customers to make reservations or purchase goods online, then the website would request bank details. Therefore, the personal data about customers (email addresses, phone numbers, postal addresses etc.) and sensitive information (bank details) must be kept secure and maybe only accessible with a password.

A large security problem related to user input is SQL Injection. The attacker uses input fields to enter code which can alter the database. To prevent this, all user input should be validated (the current version of the website only has some basic validation).

If we were to make the website as secure as possible, we could consider encrypting user information, such as credit card details. Without encrypting such data, this type of information is sent as a plain text, exactly how it was entered by the user.

Last but not least, the most important yet common mistake when it comes to most Internet users is not using a sophisticated password (including lowercase/uppercase characters, numbers, special characters etc.). Therefore, users must be careful when choosing their passwords and they must not share such information with anyone.
